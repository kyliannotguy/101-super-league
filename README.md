# 一零一超级联赛管理与展示系统

本项目是一个初版静态网页应用，用于管理和展示一零一超级联赛数据。

## 本地运行

```bash
python3 -m http.server 3000
```

然后打开：

```text
http://localhost:3000/
```

## 用 VS Code 查看

可以直接打开工作区文件：

```text
101-super-league.code-workspace
```

## 初始管理员

```text
账号：rxnb
密码：由项目维护者掌握，源码中只保留哈希
```

管理员信息可以在“赛季管理”里修改，用于后续交接。

## 已有功能

- 公开展示模式和管理员模式
- 管理员专用浏览量统计：公开展示模式不显示，同一浏览器会话计一次访问
- 五支初始球队：101fc、101竞技、格调、poi、chasing
- 球队颜色和当前赛季初始丸数设置
- 球员录入、编辑、Excel/CSV 名单导入、门将标记、队长标记、档位颜色区分
- 比赛录入、比分录入、选择球员录入进球、预计赛程
- 联赛积分榜自动统计
- 射手榜自动统计，支持管理员手动补录和撤回补录记录
- 出场成本自动计入财政：首发 4 丸，替补 2 丸
- 正常转会、解约金、自由签约、临时租借、手动调整
- 临时租借固定扣借入队 8 丸，出租队收入 4 丸，剩余 4 丸直接扣除；球员会临时显示在借入队并标记“租借”，借入队完成一场比赛后自动归还
- 转会价格自动进入财政流水
- 手动财政调整：奖金、罚款、补助、特殊调整
- 比赛、转会、手动财政流水撤回
- 赛季归档和新赛季开启
- 数据导入导出备份

## 球员名单导入格式

管理员进入“球队球员”页后，可以上传 `.xlsx`、`.xls` 或 `.csv` 球员名单。第一行需要是表头，支持这些列名：

```text
姓名、球队、位置、档位、队长
```

示例：

```text
姓名,球队,位置,档位,队长
张三,101fc,GK,顶星档,否
李四,格调,非门将,普通档,是
```

重复姓名会自动更新球员信息，不重复则新增球员。

如果一个工作簿里按球队分成多个工作表，工作表名称可以直接写球队名；这样每张表里可以省略“球队”列。

## 当前限制

当前版本会优先同步到 Supabase 线上数据库，并保留浏览器本地缓存作为兜底。管理员登录仍是轻量前端模式，适合校内联赛快速使用；如果未来需要更强安全性，可以再升级成服务端权限校验。
